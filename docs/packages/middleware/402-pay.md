# @bsv/402-pay

Client-side HTTP 402 payment handler — automatically pay 402 responses using a BRC-100 wallet.

## Install

```bash
npm install @bsv/402-pay
```

## Quick start

```typescript
import { PaymentHandler } from '@bsv/402-pay';
import { WalletToolbox } from '@bsv/wallet-toolbox';

const wallet = new WalletToolbox();
const handler = new PaymentHandler({
  wallet: wallet
});

// Make request with automatic 402 handling
const response = await handler.fetch('https://api.example.com/premium/data');

if (response.status === 200) {
  const data = await response.json();
  console.log('Data received:', data);
}
```

## What it provides

- **Automatic 402 handling** — Detect and respond to 402 responses
- **Invoice parsing** — Extract payment details from 402 response
- **Payment signing** — Sign transactions with wallet
- **Transaction broadcasting** — Submit payment transaction
- **Receipt management** — Store and verify payment receipts
- **Retry logic** — Retry request after payment
- **Timeout handling** — Wait for payment with timeout
- **Multiple payment types** — Support various invoice formats

## When to use

- Building clients that interact with paid APIs
- Creating applications that need to pay for content on-demand
- Implementing automatic payment for API access
- Building smart clients that handle 402 gracefully
- Testing payment-gated services

## When not to use

- For server-side payment verification — use @bsv/payment-express-middleware
- If you want manual payment control — don't use auto-handler
- For subscription-based access — use authentication instead
- For large bulk payments — use dedicated payment processing

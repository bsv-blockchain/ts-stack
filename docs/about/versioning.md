# Versioning

ts-stack follows Semantic Versioning (MAJOR.MINOR.PATCH) for all packages.

## Version Format

- **MAJOR** — Breaking changes (incompatible API changes)
- **MINOR** — New features (backward compatible)
- **PATCH** — Bug fixes (backward compatible)

## Release Cadence

- **Stable packages** — Monthly or as-needed for bugs
- **Beta packages** — May have breaking changes
- **Experimental packages** — Expect significant changes

## Support

- Latest major version receives bug fixes
- Previous major version receives critical security fixes only
- Older versions are unsupported

## API Stability

Check each package's documentation for:
- Current version and stability status
- Breaking changes since last version
- Deprecation notices

## Changelog

See the [GitHub repository](https://github.com/bsv-blockchain/ts-stack/releases) for detailed changelogs.
